# Place Model Testing Documentation

## Overview
This file is comprehensive testing documentation for the Place model in the HBnB application. The testing covers validation logic, API endpoints, unit tests, and integration tests using various tools including cURL, pytest, and Swagger documentation validation.

## Place Model Specifications

### Model Structure
- **Inheritance**: Inherits from BaseModel (provides id, created_at, updated_at)
- **Required Fields**: title, price, latitude, longitude, owner_id
- **Optional Fields**: description
- **Relationships**: reviews (list), amenities (list)

### Validation Rules
1. **Title**: Required, string, maximum 100 characters
2. **Description**: Optional, string if provided
3. **Price**: Required, number (int/float), must be positive
4. **Latitude**: Required, number, must be between -90.0 and 90.0
5. **Longitude**: Required, number, must be between -180.0 and 180.0
6. **Owner**: Required, must reference a valid user ID
7. **Amenities**: List of valid amenity IDs
